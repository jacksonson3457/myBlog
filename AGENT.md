# AGENT.md

このファイルは、このリポジトリで作業する「人間開発者 / AI エージェント」共通の運用ガイドです。

## 目的

- 変更の一貫性を保つ
- 既存実装への影響を最小化する
- ドキュメントと実装の同期を保つ

## 作業開始前チェック

1. ルート `README.md` を確認する
2. `docs/adr/` の既存判断を確認する
3. 変更対象が `my-blog/` かインフラ設定かを切り分ける

## コーディング方針

- TypeScript の型安全性を優先する
- 既存の命名規則・フォルダ構成に合わせる
- 無関係なリファクタを混ぜない
- コメントは「なぜ必要か」を短く書く

## テスト方針

変更時は、影響範囲に応じて以下を実行する。

```bash
cd my-blog
npm test
npm run lint
```

ビルドや配備経路に影響する場合:

```bash
cd my-blog
npm run build
```

## ドキュメント更新ルール

以下に該当する変更を加えた場合は、ドキュメントも同時更新する。

- セットアップ手順の変更: `README.md`
- 設計上の重要判断: `docs/adr/*.md`
- 運用ルールの変更: `AGENT.md`

## ADR 運用

- 保存先: `docs/adr/`
- 命名: `NNNN-<short-title>.md`（例: `0002-image-strategy.md`）
- ステータス: `Proposed` / `Accepted` / `Superseded`
- 既存判断を上書きする場合は、新しい ADR で差分理由を明記する

## コミット指針

- 1コミット1目的を意識する
- 件名は命令形で簡潔に書く
- ドキュメント変更のみでもコミットを分けてよい
